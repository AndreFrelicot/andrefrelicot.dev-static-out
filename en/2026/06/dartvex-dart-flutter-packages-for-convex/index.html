Dartvex: Dart and Flutter packages for Convex

Published Jun 19, 2026
·3 min read

Dartvex is a set of Dart and Flutter packages for using Convex from the Dart ecosystem.

The goal is simple: make Convex usable in Dart and Flutter projects without waiting for an official SDK. The package family covers the main needs of a real app: realtime subscriptions, queries, mutations, actions, Flutter integration, typed bindings, authentication and offline-friendly patterns.

Dartvex is not an official Convex project. It is an independent release, published on pub.dev under my verified publisher account.

Why Dartvex exists

Convex already has strong first-party support for JavaScript and React environments. Flutter teams, however, often need the same kind of realtime backend experience from Dart.

That demand is visible on the Convex public roadmap: the request for an official Flutter/Dart SDK is heavily backed, and it asks for a pure Dart client, realtime subscriptions, authentication, optimistic updates, offline-friendly patterns and code generation.

Convex has also indicated in that same request that a third official implementation of the client-side state machine is extremely unlikely. In practice, that means Flutter and Dart developers should not expect a first-party Dart SDK to appear.

Dartvex exists for that gap: a Dart-native ecosystem for teams that want Convex in Flutter or Dart today.

The package ecosystem

The first public release was published on June 19, 2026. The current ecosystem is split into five packages, so developers can install only the layers they need.

PackageRole
dartvexCore Dart client for Convex: realtime sync, subscriptions, queries, mutations, actions, authentication and Convex values.
dartvex_flutterFlutter integration layer with provider-style setup and reactive widgets for queries and mutations.
dartvex_codegenCommand-line generator that creates type-safe Dart bindings from a Convex backend schema.
dartvex_localOffline-capable extension with a local SQLite cache and a mutation queue for mobile-first workflows.
dartvex_auth_betterAdapter for using Better Auth with Dartvex and Convex.

How to choose the packages

For a Flutter app, the usual starting point is dartvex_flutter. It brings the UI layer and depends on the core dartvex package.

For a pure Dart application, command-line tool or server-side Dart project, dartvex is the base package.

When the backend schema should become a typed Dart API, dartvex_codegen adds the generation step.

When the product needs to keep working with unstable connectivity, dartvex_local adds the local-first layer.

When the project uses Better Auth, dartvex_auth_better provides the authentication adapter.
